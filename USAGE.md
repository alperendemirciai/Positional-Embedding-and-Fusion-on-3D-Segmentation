## 1. Setup

### Requirements

- Python 3.9+
- CUDA-capable GPU with ≥8GB VRAM (≥16GB recommended for V2 models)
- ~30GB free disk space for dataset + checkpoints + predictions

### Installation

```bash
pip install -r requirements.txt
```

All MONAI extras (nibabel, tqdm) are included via `monai[nibabel,tqdm]`.

### Dataset Layout

Place the BraTS 2021 data so the directory structure matches:

```
dataset/
├── BraTS2021_Training_Data/
│   ├── BraTS2021_00000/
│   │   ├── BraTS2021_00000_t1.nii.gz
│   │   ├── BraTS2021_00000_t1ce.nii.gz
│   │   ├── BraTS2021_00000_t2.nii.gz
│   │   ├── BraTS2021_00000_flair.nii.gz
│   │   └── BraTS2021_00000_seg.nii.gz
│   ├── BraTS2021_00001/
│   │   └── ...
│   └── ...   (1251 patient folders total)
├── BraTS2021_00495/   (sanity-check case, not used in splits)
└── BraTS2021_00621/   (sanity-check case, not used in splits)
```

## 2. Generating Data Splits

Run once before any training. Produces `data_utils/splits.json` with a reproducible 70/15/15 patient-level split.

```bash
python data_utils/split_dataset.py
```

Optional arguments:

```bash
python data_utils/split_dataset.py \
    --data_root dataset/BraTS2021_Training_Data \   # default
    --output    data_utils/splits.json \            # default
    --seed      42                                  # default
```

Expected output:
```
Total patients : 1251
Train          : 875
Val            : 188
Test           : 188
Splits saved to data_utils/splits.json
```

The split file is a simple JSON:
```json
{
  "train": ["BraTS2021_00042", "BraTS2021_00100", ...],
  "val":   [...],
  "test":  [...]
}
```

**Do not regenerate the splits after training has started** — the split must remain fixed across all experiments for a fair comparison.

## 3. Configuration System

Every runtime parameter is controlled through YAML files. `train.py` and `evaluate_all.py` deep-merge your experiment config on top of `configs/base_config.yaml`, so experiment configs only need to specify the keys that differ.

### Full `base_config.yaml` Reference

```yaml
data:
  root: "dataset/BraTS2021_Training_Data"  # path to the dataset root
  splits_file: "data_utils/splits.json"    # generated by split_dataset.py
  patch_size: [96, 96, 96]                 # training patch size (H, W, D)
  num_workers: 4                           # DataLoader worker processes
  cache_rate: 0.0                          # 0=no cache; 1.0=full RAM cache (~30GB)
  foreground_ratio: 0.5                    # fraction of patches centred on tumour

model:
  variant: "V0EarlyFusion"   # V0EarlyFusion | V1SharedBackbone | V2SeparateBackbones
  base_channels: 16          # first encoder level channels; doubles each level
  depth: 4                   # encoder levels; bottleneck = depth+1
  bottleneck_channels: 256   # must equal base_channels * 2^depth

fusion:
  strategy: "mean"           # mean | weighted | attention  (V1/V2 only)

training:
  max_epochs: 300
  batch_size: 2              # use 1 for V2 to stay within 16GB VRAM
  lr: 1.0e-4
  weight_decay: 1.0e-5
  grad_clip: 1.0
  early_stopping_patience: 50
  mixed_precision: true      # FP16 via torch.cuda.amp — strongly recommended
  num_patches_per_sample: 2  # patches sampled per patient per epoch
  loss: "dice_ce"
  loss_dice_weight: 0.5      # dice fraction in DiceCE (BCE = 1 - this)

pe:
  type: "none"               # none | film | concat
  sinusoidal_num_freqs: 5    # Fourier frequencies → 30-dim PE vector
  film_hidden_dim: 128       # hidden size of FiLM MLP

inference:
  sw_batch_size: 4           # parallel patches in SlidingWindowInferer
  sw_overlap: 0.5            # overlap between adjacent windows (0.25–0.75)
  sw_mode: "gaussian"        # gaussian | constant
  threshold: 0.5             # sigmoid threshold for binary mask
  save_predictions: true     # write per-patient .nii.gz to results/predictions/

evaluation:
  compute_hd95: true         # also compute 95th-percentile Hausdorff distance
  missing_modality: false    # run all 15 modality-subset evaluations (V1/V2)

logging:
  tensorboard_dir: "experiments/logs"
  checkpoint_dir: "checkpoints"
  results_dir: "results"
  log_every_n_epochs: 1
  save_every_n_epochs: 50    # periodic checkpoint interval
```

### Writing a Custom Experiment Config

Only specify the keys you want to override:

```yaml
# configs/my_experiment.yaml
model:
  variant: "V1SharedBackbone"
  base_channels: 32          # larger network
pe:
  type: "film"
fusion:
  strategy: "weighted"
training:
  batch_size: 2
  lr: 5.0e-5                 # lower learning rate
evaluation:
  missing_modality: true
```

## 4. Training

### Single Experiment

```bash
python train.py \
    --config          configs/v0_nope.yaml \
    --experiment_name v0_nope
```

| Argument | Required | Description |
|----------|----------|-------------|
| `--config` | Yes | Path to experiment YAML config |
| `--experiment_name` | Yes | Unique name — used for checkpoint and log filenames |
| `--resume` | No | Path to a checkpoint to continue training from |
| `--seed` | No | Random seed (default: 42) |

### Resuming a Run

```bash
python train.py \
    --config          configs/v2_film.yaml \
    --experiment_name v2_film \
    --resume          checkpoints/v2_film_epoch050.pth
```

### Checkpoint Files

Saved to `checkpoints/`:

| File | Saved when |
|------|-----------|
| `{name}_best.pth` | Whenever a new best validation Dice is achieved |
| `{name}_epoch{NNN}.pth` | Every `save_every_n_epochs` epochs (default: 50) |

Each checkpoint contains: `epoch`, `state_dict`, `optimizer`, `scheduler`, `best_val_dice`, `cfg`.

### What to Watch During Training

The console prints one line per epoch:
```
Epoch 042/300 | Loss 0.3821 | Val WT 0.881 TC 0.812 ET 0.743 Mean 0.812 | 127s
  → New best val Dice: 0.8120
```

For live curves, use TensorBoard (see [Section 9](#9-tensorboard)).

### Memory Troubleshooting

If you hit OOM:

1. Reduce `training.batch_size` to 1
2. Reduce `inference.sw_batch_size` to 2 or 1
3. Reduce `data.cache_rate` to 0.0
4. Reduce `model.base_channels` from 16 to 12
5. Reduce `data.patch_size` from [96,96,96] to [64,64,64]

## 5. Evaluation

### Evaluate All Experiments

```bash
python evaluate_all.py
```

This iterates over all 13 experiment configs, loads `checkpoints/{name}_best.pth`, and runs full-volume sliding-window inference on the 188 test patients.

### Evaluate a Subset

```bash
python evaluate_all.py --configs v0_nope v1_film v2_nope_mean
```

### What Gets Saved

For each experiment:

```
results/
├── {exp_name}_test_results.json      # per-patient and aggregate scores
└── predictions/
    └── {exp_name}/
        ├── BraTS2021_01200_pred.nii.gz
        ├── BraTS2021_01201_pred.nii.gz
        └── ...
        └── missing_modality/          # V1/V2 only
            ├── t1/
            ├── t1+t2/
            └── ...
```

### Results JSON Format

```json
{
  "per_patient": {
    "BraTS2021_01200": {"WT": 0.912, "TC": 0.841, "ET": 0.783},
    "BraTS2021_01201": {"WT": 0.887, "TC": 0.803, "ET": 0.741},
    ...
  },
  "mean": {"WT": 0.891, "TC": 0.814, "ET": 0.762},
  "std":  {"WT": 0.051, "TC": 0.083, "ET": 0.112},
  "hd95_mean": {"WT": 4.21, "TC": 6.87, "ET": 3.14},
  "missing_modality": {
    "t1":         {"WT": 0.831, "TC": 0.741, "ET": 0.612},
    "t1ce":       {"WT": 0.856, "TC": 0.792, "ET": 0.751},
    "t1+t1ce":    {"WT": 0.871, "TC": 0.805, "ET": 0.762},
    ...
  }
}
```

### Missing Modality Subsets

All 15 non-empty subsets tested for V1 and V2:

| Size | Subsets |
|------|---------|
| 1 | t1, t1ce, t2, flair |
| 2 | t1+t1ce, t1+t2, t1+flair, t1ce+t2, t1ce+flair, t2+flair |
| 3 | t1+t1ce+t2, t1+t1ce+flair, t1+t2+flair, t1ce+t2+flair |
| 4 | t1+t1ce+t2+flair (full) |

To test a specific subset manually from Python:

```python
# V1 or V2 model, active_modalities is a tuple of indices
# 0=T1, 1=T1ce, 2=T2, 3=FLAIR
logits = model(x, active_modalities=(0, 2))   # T1 + T2 only
```

## 7. Running All Experiments

### One-Shot (all 13 experiments)

```bash
bash experiments/run_all.sh
```

This will:
1. Generate `splits.json` (if not already present)
2. Train all 13 experiments sequentially
3. Skip any experiment whose `_best.pth` already exists
4. Run `evaluate_all.py`
5. Run `results_analysis.py`

### Selective Run

```bash
bash experiments/run_all.sh v0_nope v1_film v2_nope_mean
```

### Recommended Run Order

Due to training time differences, run the fast experiments first:

```bash
# Phase 1 — V0 and V1 (~6h each, ~48h total)
bash experiments/run_all.sh v0_nope v0_film v0_concat v1_nope_mean v1_nope_weighted v1_nope_attention v1_film v1_concat

# Phase 2 — V2 (~22h each, ~110h total)
bash experiments/run_all.sh v2_nope_mean v2_nope_weighted v2_nope_attention v2_film v2_concat
```

## 8. Visualising Predictions

Predicted segmentation volumes are saved as `.nii.gz` files with the same affine and header as the original T1 scan, so they are directly viewable alongside the source MRI.

### ITK-SNAP

1. File → Open Main Image → select `BraTS2021_XXXXX_t1.nii.gz`
2. Segmentation → Load From File → select `results/predictions/{exp_name}/BraTS2021_XXXXX_pred.nii.gz`
3. Label colours: 1=NCR (blue), 2=ED (yellow), 4=ET (red)

### 3D Slicer

1. File → Add Data → select all 5 files for a patient
2. The prediction file will load as a label map overlay

### Label Encoding

Predictions are saved in standard BraTS label encoding:

| Value | Region |
|-------|--------|
| 0 | Background |
| 1 | NCR — Necrotic Tumour Core |
| 2 | ED — Peritumoral Edema |
| 4 | ET — Enhancing Tumour |

## 9. TensorBoard

```bash
tensorboard --logdir experiments/logs
```

Then open `http://localhost:6006` in your browser.

Per-experiment scalar logs:

| Tag | Description |
|-----|-------------|
| `Loss/train` | Mean training loss per epoch |
| `Dice/val_WT` | Validation Dice — Whole Tumour |
| `Dice/val_TC` | Validation Dice — Tumour Core |
| `Dice/val_ET` | Validation Dice — Enhancing Tumour |
| `Dice/val_mean` | Mean of the three validation Dice scores |

To compare all experiments simultaneously, open the TensorBoard Scalars tab and enable multiple runs.

## 10. Common Issues

### `FileNotFoundError: data_utils/splits.json`

Run `python data_utils/split_dataset.py` first.

### CUDA Out of Memory during training

In your experiment YAML:
```yaml
training:
  batch_size: 1
  mixed_precision: true
```

If still OOM:
```yaml
model:
  base_channels: 12   # reduced from 16
data:
  patch_size: [64, 64, 64]   # reduced from [96, 96, 96]
```

### CUDA Out of Memory during inference

In your experiment YAML:
```yaml
inference:
  sw_batch_size: 1   # reduced from 4
```

### `cache_rate: 1.0` causes system freeze

Your system does not have enough RAM for full dataset caching (~30GB). Set:
```yaml
data:
  cache_rate: 0.0
```

Alternatively set `cache_rate: 0.25` to cache a quarter of the dataset.

### V2 training is very slow

V2 runs 4 sequential forward passes per step. This is by design to stay within VRAM. To speed it up on a multi-GPU machine, you could distribute the 4 backbones across GPUs — but this is not implemented in the current codebase.

### Missing-modality evaluation taking too long

The full 15-subset evaluation on 188 patients × 15 subsets = 2820 inference runs. To skip it:
```yaml
evaluation:
  missing_modality: false
```

Or evaluate only the full-modality result and specific subsets you care about by modifying `evaluate_all.py`.

### `monai.transforms.ConvertToMultiChannelBasedOnBratsClassesd` not found

Ensure MONAI ≥ 1.3.0 is installed:
```bash
pip install --upgrade "monai[nibabel,tqdm]>=1.3.0"
```

### Predictions look misaligned in ITK-SNAP

This should not happen since predictions reuse the original NIfTI affine. If it does, verify that the `_t1.nii.gz` file used as the reference header belongs to the same patient as the prediction.
